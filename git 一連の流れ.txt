git（バージョン管理システム） 一連の流れ

1　デスクトップにフォルダ（intro_git）を作成、その中にテキストファイル(first.txt ”gitの勉強をしていきます。”)を作成
2　作製したファイルにディレクトリ移動する　cd ~/Desktop → cd intri_git　→※中身を確認  ls
3　ローカルリポジトリを作成する　git init　※.gitフォルダが出来ているか確認　ls -a
4　first.txtにコメントを加え（"基本的なワークフローを学びます。"）保存
5　ステージングエリア（控室のようなもの）に変更点を追加する　　git add ファイル名(first.txt)　

	※一気にたくさんの変更をしてしまったとき、commitするファイルを選べるようにするのが「ステージングエリア」の役割。commit準備のための「控室」。
	  git add ファイル名で変更点追加ができるが、手持ちの変更点すべてを一度に追加したいときはgit add.(最後がピリオド)で全部追加できる


6　ローカルリポジトリにコミット（預ける・記録）する　git commit　　※１作業ごとに１commitするのが原則！これにより変更が時系列で記録される
	VIM画面に移動　　半角入力モードに変更
			一番先頭に小文字の[i]を入力　文字入力モードになる
			initial commit（今回は最初のコミットなのでこの名前を使用）と入力
			[ESC]キーを押す
			[：]+　[W]　+ [Q]　と入力　[enter]を押す
	もとの画面に戻ってくる
	※コミットはメッセージが残せるのでコミットごとに何の変更をしたかを残す
	※コミットメッセージは他のメンバーに「何をどうして変更したのか」が伝わるように書く
		１行目　：　変更内容の要約（タイトル・概要　一目ですぐにわかるものにする）
		２行目　：　空行
		３行目　：　変更した理由（変更の背景を伝える）

7　GitHUBの準備をする　T-Matsumoto-mse365のホーム画面を開いたら、猫マークの右横の＋ボタンをクリック
　　「ニューリポジトリ」をクリックする
8　[reate a new repository]画面で[Repository name]を入力（intro_git）、その下の[Description]は空欄のままにして
　　その下の　Public を選ぶ
9　一番下の[create repository]ボタンをクリックすると新しいリポジトリ画面が作成される
10　今回はすでに作成されているリポジトリに対して行うので、[...or push an sxisiting repository from the command line]の中の
　　git remote add oritin https://～（gitにgitHUBを登録するというコマンド）をコピーする
11　GitBashに戻って先ほどコピーしたものを貼り付けEnterを押すと登録される
12　登録されたら　git push -u origin master と入力し実行する
13　正常にプッシュされたらgitHUB 画面を更新して確認する
14　git statusで現在のディレクトリの状況を確認する
15　first.txtの内容を変更（今回は追加）する　git statusでステージングエリアを確認すると、一部赤字で表示、変更されたことがわかる
16　git add .(ドットの前にはスペースを入れる！)を実行し、変更点すべてをステージングエリアに追加する
17　git commit -vと入力し実行　VIM画面に移動する　※-vと打つと、今回の変更点が表示される
18　一番上の行にコメントを残す　1行目にタイトル（add,commitのコマンドを追記）
19　2行目は空行
20　3行目に理由を記載（今回は特に理由はないが、「講座3-4を受講したから」とした）　書き終わったら　[esc]+[:]+[W]+[Q]を押して  [enter]


~いろいろな変更をしたときのコミットの仕方～

git add .
git rm
git status

git log --oneline(コミットしたログを1行で表示)
git log -p index.html(ファイルの差分を表示する)
git log -n 3(最新の３つのログを表示)
git log --oneline -n 3(最新のログ３つを1行で表示)
